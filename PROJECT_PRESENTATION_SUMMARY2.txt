--------------------------------------------------------------------------------
1. PROJECT OVERVIEW
--------------------------------------------------------------------------------
DermaVision is an AI-assisted dermatology web platform that helps users
understand potential skin conditions through image-based analysis and a safe
educational chatbot.

Key Points:
  - Users upload a skin image and receive an AI-based preliminary diagnosis
  - A RAG-powered chatbot answers questions about the detected condition
  - Printable reports are available for educational or consultation purposes
  - NOT a final medical diagnosis — dermatologist confirmation is always required
  - Designed as an assistive/educational tool, not a replacement for medical care


--------------------------------------------------------------------------------
2. MAIN USER FLOW
--------------------------------------------------------------------------------
  1. User signs up / logs in (JWT-secured authentication)
  2. User uploads a skin image (JPG/JPEG/PNG, max 10 MB)
  3. Backend validates the image and runs ML inference
  4. ML model returns: predicted condition, confidence score, top predictions,
     and image quality score
  5. User sees the diagnosis result page

  From the result page, the user can:
    a) Download a Preliminary Report based only on the AI diagnosis result
    b) Open the RAG Chatbot to ask questions about the condition

  6. Chatbot answers questions such as:
       - "What is this disease and is it dangerous?"
       - "Is it contagious? What should I do?"
       - "Can children get it? Should I see a doctor?"
       - "What are general care and treatment guidelines?"

  7. After chatting, the user can generate a Consultation Report summarizing
     the conversation context, concerns, and general guidance
  8. User can view diagnosis history and manage family member profiles


--------------------------------------------------------------------------------
3. ARCHITECTURE / STRUCTURE
--------------------------------------------------------------------------------
Full-Stack Project:
  - Frontend: React + Vite (TypeScript)
  - Backend:  FastAPI (Python), async architecture

Backend — Layered / Clean Architecture:
  - API Layer          : Endpoints, request/response handling, schemas
  - Domain/Service Layer: Business logic (diagnosis, RAG, reports, auth)
  - Infrastructure Layer: Database, ML model, RAG engine, email, LLM providers
  - Database Layer     : SQLAlchemy async ORM models and repositories

Frontend Structure:
  - Pages, Components, Services, Contexts
  - Localization (Arabic / English)
  - API configuration and auth context


--------------------------------------------------------------------------------
4. DESIGN PATTERNS USED
--------------------------------------------------------------------------------
  - Layered / Clean Architecture    : Clear separation of concerns
  - Service Layer Pattern           : Encapsulated business logic
  - Repository Pattern              : Abstracted database access
  - DTO / Mapper Pattern            : Consistent, clean API responses
  - Provider / Strategy Pattern     : Switch between Groq, xAI, OpenAI, or local
  - Fallback / Resilience Pattern   : Safe deterministic fallback if LLM fails
  - Centralized Configuration       : All settings via environment variables


--------------------------------------------------------------------------------
5. AI DIAGNOSIS
--------------------------------------------------------------------------------
Model:
  - EfficientNet-B4 (PyTorch / timm)
  - Trained for skin condition classification

Supported Classes:
  - Urticaria
  - Tinea Circinata
  - Annular Lichen / Lichen Planus
  - Psoriasis
  - Mycosis Fungoides (MF)
  - Healthy

Model Output:
  - Predicted label
  - Confidence score
  - Top-N predictions
  - Image quality score


--------------------------------------------------------------------------------
6. RAG CHATBOT
--------------------------------------------------------------------------------
  - Retrieval-Augmented Generation grounded in local dermatology knowledge file
    (skin_data1.txt) — grounded in local dermatology knowledge with deterministic safety rules
  - Supports Arabic and English
  - Handles multi-intent questions in a single query
  - LLM providers: Groq / OpenAI / xAI (configurable), with local fallback
  - Sources can be shown in the UI for transparency
  - Chat history is saved per diagnosis session (browser session storage)
  - Answers cover: disease overview, danger level, contagion, treatment
    guidance, age groups (children/adults), and when to visit a doctor


--------------------------------------------------------------------------------
7. MEDICAL SAFETY GUARDRAILS
--------------------------------------------------------------------------------
  - No personal prescriptions generated
  - No exact dose, frequency, or duration provided
  - Urgent-care detection flags dangerous symptoms
  - Deterministic safety profiles per disease
  - Urticaria  → forced non-contagious regardless of LLM output
  - Tinea      → forced contagious regardless of LLM output
  - Canonical disease name mapping prevents incorrect translations
  - Post-generation validation rejects unsafe LLM answers
  - Disclaimer shown to all users on every result and report


--------------------------------------------------------------------------------
8. REPORT SYSTEM
--------------------------------------------------------------------------------
Two separate report flows:

  a) Preliminary / Basic Diagnosis Report
       - Generated directly from the AI diagnosis result page
       - Downloadable and printable
       - Based solely on ML prediction and general guidance
       - No chat required

  b) Consultation Report
       - Generated after the RAG chatbot conversation
       - Includes: diagnosis context, summary of patient concerns,
         general care guidance, red flags, and disclaimer
       - Available as modal / printable / copyable

Important:
  - Neither report is a final medical diagnosis
  - Neither report is a personal treatment prescription
  - Both reports are labeled clearly as "Preliminary" or "Consultation"


--------------------------------------------------------------------------------
9. SECURITY FEATURES
--------------------------------------------------------------------------------
  - JWT-based authentication with protected endpoints
  - User ownership verification for all diagnosis and report access
  - Password hashing (bcrypt)
  - Password reset via hashed, single-use tokens
  - Image upload validation: JPG/JPEG/PNG only, 10 MB limit
  - CORS configuration
  - Security headers middleware
  - All secrets stored in backend .env (ignored by Git)
  - .env.example contains placeholders only — no real credentials

14. SHORT PRESENTATION SUMMARY (for slides / viva)
--------------------------------------------------------------------------------
DermaVision is a full-stack AI-assisted dermatology system. It combines
image-based skin disease prediction, secure user management, diagnosis history,
a safe RAG medical assistant, and printable consultation reports. The backend
follows layered clean architecture with service, repository, DTO, and provider
strategy patterns. The system supports Arabic and English, offers Docker-based
setup, and enforces medical safety guardrails to ensure all output remains
educational and is never presented as a personal prescription or final diagnosis.
